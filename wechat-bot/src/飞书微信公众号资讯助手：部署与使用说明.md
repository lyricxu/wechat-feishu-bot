# 飞书微信公众号资讯助手：部署与使用说明

## 1. 引言

本说明文档旨在指导用户部署和使用基于 Python 开发的飞书微信公众号资讯助手。该机器人能够定时或手动爬取指定微信公众号的最新文章，利用人工智能（AI）自动阅读文章并生成概要、进行质量打分，并通过飞书消息推送给用户。同时，对于高分文章，机器人会自动将其存储到用户指定的 Notion 数据库中，实现资讯的自动化收集、筛选、总结与归档。

## 2. 功能概览

*   **微信公众号文章爬取**：根据公众号名称定向获取最新文章链接及内容。
*   **AI 智能分析**：
    *   自动生成文章概要。
    *   基于信息密度、逻辑严密性和行业相关度对文章质量进行 0-100 分的评分。
    *   提取文章关键词。
*   **飞书消息推送**：将分析结果（标题、概要、评分、关键词、原文链接）以富文本形式发送至指定飞书用户或群聊。
*   **Notion 数据库存储**：自动将达到高分阈值的文章信息（标题、概要、评分、链接、关键词、作者）保存至 Notion 数据库。
*   **自动化与手动触发**：支持定时任务自动运行，也可手动触发执行。

## 3. 环境准备

在部署机器人之前，请确保您的系统满足以下要求并完成相关凭证的获取。

### 3.1 软件环境

*   **Python 3.8+**：推荐使用最新稳定版本。
*   **Playwright**：用于模拟浏览器行为，爬取微信公众号文章。

### 3.2 平台凭证获取

#### 3.2.1 飞书机器人凭证

1.  访问 [飞书开放平台](https://open.feishu.cn/app?lang=zh-CN)。
2.  点击“创建自建应用”，填写应用名称（例如：“资讯助手”）、应用描述等信息。
3.  在左侧导航栏选择 **凭证与基础信息**，复制您的 **App ID** 和 **App Secret**。这些将用于机器人与飞书 API 的认证。
4.  在 **应用功能 -> 机器人** 中点击“启用机器人”。
5.  在 **权限管理** 中搜索并勾选 `im:message:send_as_bot` (以应用的身份发送消息) 权限，并发布一个版本（或点击“申请开通”）。
6.  获取您的飞书用户 `open_id` 或群聊 `chat_id`，这将是机器人推送消息的目标 ID。您可以通过飞书开放平台的 [获取用户 ID](https://open.feishu.cn/document/home/user-identity-introduction/how-to-get-user-id) 文档或使用飞书提供的工具获取。

#### 3.2.2 Notion API 凭证

1.  访问 [Notion My Integrations](https://www.notion.so/my-integrations)。
2.  点击“New integration”，命名为“News Bot”或其他您喜欢的名称，并选择关联的工作区。
3.  复制生成的 **Internal Integration Token**。这是机器人访问 Notion API 的密钥。
4.  在 Notion 中创建一个新的数据库（Database），用于存储文章信息。数据库中应包含以下属性（属性名称需与代码中的保持一致，类型如下）：
    *   **标题** (Title)
    *   **概要** (Text)
    *   **分数** (Number)
    *   **链接** (URL)
    *   **标签** (Multi-select)
    *   **作者** (Select)
5.  将您创建的 Integration (News Bot) 连接到该数据库。在 Notion 数据库页面，点击右上角的 `...` -> `Add connections`，搜索并选择您的 Integration。
6.  复制该 Notion 数据库的 URL。URL 中 `notion.so/` 后面到 `?` 之前的那串 32 位字符就是 **Database ID**。

## 4. 项目部署

### 4.1 获取项目代码

请将项目代码下载到您的本地环境。假设项目根目录为 `/home/ubuntu/wechat_bot`。

### 4.2 安装依赖

进入项目根目录，安装 Python 依赖：

```bash
cd /home/ubuntu/wechat_bot
pip install -r requirements.txt
```

### 4.3 安装 Playwright 浏览器驱动

由于微信公众号文章爬取需要模拟浏览器环境，请安装 Playwright 驱动：

```bash
playwright install chromium
```

### 4.4 配置 `config.yaml`

打开项目根目录下的 `config/config.yaml` 文件，填入您在 `3.2 平台凭证获取` 步骤中获得的凭证信息和配置项：

```yaml
# 飞书机器人配置
feishu:
  app_id: "YOUR_APP_ID" # 替换为您的飞书 App ID
  app_secret: "YOUR_APP_SECRET" # 替换为您的飞书 App Secret

# Notion 配置
notion:
  token: "YOUR_NOTION_TOKEN" # 替换为您的 Notion Internal Integration Token
  database_id: "YOUR_DATABASE_ID" # 替换为您的 Notion 数据库 ID

# 微信公众号配置
wechat:
  # 待监控的公众号列表，请替换为实际的公众号名称
  target_accounts:
    - "公众号名称1"
    - "公众号名称2"
  # 爬取间隔（秒），建议不要设置过短，避免触发反爬机制
  check_interval: 3600 # 示例：每小时检查一次

# AI 打分与概要配置
ai:
  model: "gpt-4.1-mini" # 使用的 AI 模型，可根据实际情况调整
  # 高分阈值，超过此分数的文章存入 Notion
  high_score_threshold: 80 
  # 评分维度权重，可根据您的偏好调整
  weights:
    depth: 0.4 # 信息深度
    logic: 0.3 # 逻辑严密性
    relevance: 0.3 # 行业相关度

# 存储路径
storage:
  data_dir: "./data"
  log_dir: "./logs"
```

**重要提示**：
*   `feishu.app_id` 和 `feishu.app_secret` 必须填写正确。
*   `notion.token` 和 `notion.database_id` 必须填写正确。
*   `wechat.target_accounts` 填写您希望监控的微信公众号的**准确名称**。
*   `main.py` 中的 `self.feishu.send_rich_text` 方法的 `receive_id` 参数需要替换为您的飞书用户 `open_id` 或群聊 `chat_id`。

## 5. 运行机器人

### 5.1 手动运行

在项目根目录下执行 `main.py`，机器人将执行一次完整的爬取、分析、推送和存储流程：

```bash
python main.py
```

### 5.2 设置定时任务

机器人内置了 `schedule` 库，可以方便地设置定时任务。在 `main.py` 中，`start_scheduler` 方法已经配置了每天早上 9 点和每隔 4 小时运行一次任务。您可以根据需要修改 `start_scheduler` 方法中的 `schedule.every().day.at("09:00").do(self.run_task)` 和 `schedule.every(4).hours.do(self.run_task)` 来调整定时策略。

要启动定时任务，请运行：

```bash
python main.py
```

**注意**：为了让定时任务持续运行，您可能需要将此脚本部署到服务器上，并使用 `nohup` 或 `supervisor` 等工具使其在后台运行。

## 6. 核心模块说明

*   `src/scraper.py`：负责微信公众号文章的爬取。目前采用 Playwright 模拟浏览器行为，但由于微信反爬机制复杂且多变，建议在生产环境考虑接入更稳定的第三方微信文章采集 API 服务。
*   `src/ai_analyzer.py`：负责调用 AI 模型（如 GPT-4.1-mini）对文章进行概要总结、关键词提取和质量打分。
*   `src/feishu_bot.py`：负责与飞书 API 交互，获取 Access Token 并发送富文本消息。
*   `src/notion_client.py`：负责与 Notion API 交互，将高分文章信息写入指定的 Notion 数据库。
*   `main.py`：主程序入口，整合上述所有模块，控制任务流程和定时调度。

## 7. 注意事项与免责声明

*   **微信爬取限制**：微信公众号的反爬机制非常严格，任何非官方授权的爬取行为都可能面临封禁风险。本项目提供的爬取模块仅为示例，在实际使用中可能会遇到稳定性问题。强烈建议用户寻找合法合规的第三方 API 服务进行微信文章的获取。
*   **AI 成本**：AI 模型调用会产生费用，请注意您的 API 密钥余额。
*   **Notion 数据库结构**：请确保您的 Notion 数据库属性名称与代码中定义的完全一致，否则会导致写入失败。
*   **伦理与合规**：请确保您的使用行为符合微信、飞书、Notion 的服务条款以及相关法律法规，不得用于任何非法用途。

## 8. 故障排除

*   **飞书消息未发送**：检查 `config.yaml` 中的 `app_id` 和 `app_secret` 是否正确，以及飞书应用是否已启用机器人功能并拥有 `im:message:send_as_bot` 权限。同时检查 `receive_id` 是否正确。
*   **Notion 记录未添加**：检查 `config.yaml` 中的 `token` 和 `database_id` 是否正确，以及 Notion Integration 是否已连接到目标数据库，并且数据库属性名称与代码一致。
*   **文章爬取失败**：这通常是由于微信反爬机制导致。尝试调整 `check_interval`，或考虑更换更稳定的爬取方案。
*   **AI 分析失败**：检查您的 `OPENAI_API_KEY` 环境变量是否设置正确，以及 AI 模型是否可用。

如果您在部署或使用过程中遇到任何问题，请根据以上说明进行排查。如果问题依然存在，请提供详细的错误日志以便进一步协助。
